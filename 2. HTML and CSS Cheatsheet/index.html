Basic Sytax

Elements

An element consists of content between an opening and closing tags (see Figure 1 below).

Elements can be either block-level or inline-level. An block element is visually separate from the surrounding elements. An inline element can be placed within block elements without disrupting the flow of the page. Let's analyze this page to explain the difference. Notice how this block of text is separated from the text above and below it. This is the case because each paragraph is written in a separate p element, which is a block element. However, within this paragraph, the words block-level and inline-level are italicized. This was done by placing each word within the em tag, an inline element.

Some tags, such as br / are self closing and do not require a closing tag. These are known as void elements (br / and br are both valid).

Attributes

Many tags also have attributes where the developer can specify a value (see Figure 2). For example, the image tag img / has a required attribute "src" (source). The associated value is a link to the image to display. This is an example of a required attribute, but many are optional.

There are two commonly used attributes known as global attributes. These serve a very important purpose when styling a website with CSS.

  • iduniquely identifies a tag (can only be applied to 1 tag)
  • classidentifies multiple tags (can be applied to unlimited tags)

Tables

A table is initialized with a table element. The contents of this tag are split into 3 sections: thead (table head), tbody (table body), and tfoot (table foot).

The thead element contains a single tr (table row). Within this row, is a th (table header) representing each column in the table.

The tbody contains each of the rows (tr). Each row contains a td (table data) corresponding with each column.

The tfoot is optional and contains a final row. For exmple, this is useful if the table has a row displaying the sum of each column.

Lists

Lists can be of 3 types of lists: ul (unordered lists), ol (ordered lists), and dl (description lists).

The first two contain a li (list item) element for each item of the list. The description lists contain a dt (description term) and dd (description details) elements for each item in the list.

Page Structure

Elements are organized in a hierarchical structure and distinguished by level of indentation. Indentation is commonly either 2 or 4 spaces.

Each HTML document must have the statement, !DOCTYPE HTML at the top. Additionally, everything must be within an html element.

An html file will genrally be split into 2 parts: head and body. The head element contains critical information such as the title of the website and links to the CSS document. Within the body, there are usually 3 sections: header, main, and footer. The header contains information at the top of the document such as navigation links or the title. The body contains the main section of the document. The footer contains information at the bottom of the pages such as contact details.

Linking to CSS and HTML

While styling can be done directly in the HTML file, the easiest way to style an HTML document is to use a CSS file and link it in the HTML file.

To do this, add a link element in the file's head element. Within the link, define 2 attributes: href and rel. Set href to the relative link of the CSS file. Set rel to "stylesheet". This tells the browser that this element contains the link to the CSS stylesheet.

Basic Syntax

A CSS file is comprised of a list of rulesets. Each ruleset beings with a selector. This determines what elements will be affected by the style below. Within the curly brackets following the selector is known as the declaration. The declaration consists of a key-value pair. The property defines what visual aspect will be changed and is followed by a ":". The property value defines the details of the change and is followed by a ";".

Selectors

To apply a property to the entire document, set the selector of a ruleset to *. Narrowing down the search, set the selector to a tag name such as h1 to apply the changes to all the h1 elements. One can also target changes using the global attributes. To select an id, prepend a "#" to the selector. #biography will apply changes to the element with the "biography" id. To select a class, prepend a "." to the selector. .main-topics will apply the styling to all the elements containing the "main-topics" class attribute.

To target a more specific group of elements, one can combine selectors with a space. .main-topics h2 will apply the styling to all h2 elements under the element containing the "main-topics" class. If an element is targeted by multiple selectors, the most specific one will apply. For example, .main-topics h2 is more specific (thus taking priority) than h2, .main-topics, and *.

Box Model

The flow of the page is defined by the box model. Each block of content has 4 properties that effect its size: content, padding, border, and margin. The content is defined by what the element is made of (e.g. text or an image). The padding is the area between the content and the border. The border is a box around the content (often visible or size of 0). The margin is the area between outside the border. The combination of these 4 determines the size and shape of an element.

The content can be adjusted using the width and height properties, often expressed in px (pixels). The padding is affected by the padding property, the border by the border property, and the margin by the margin property. padding and margin accept 4 different values-top, right, bottom, left. These must be in order and are separated by spaces. If only given 1 value, that will be the size for all 4 directions. If given 2, the first will set the top and bottom and the second will set the left and right. If given 3, the first will set the top, the second will set the left and right, and the third will set the bottom. border accepts up to 3 values. The color, style, and width can be set in any order.

The overflow property can be optionally set to change how the browser display data that extends beyond its box. By default content is displayed, but this results in overlap. One could simply make the box bigger or set the overflow to scroll to remove overlap and allow users to view scroll through the overflowing content.
